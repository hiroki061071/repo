# アメダス連続無日照時間算出ツール

PySide6 ベースの Windows デスクトップアプリです。  
気象庁 CSV を読み込み、地点別の連続無日照時間を算出します。

## 判定ルール

- 1日の日照時間合計が **3.0 時間未満** の日を **無日照日** とする
- 連続無日照時間 = 連続無日照日数 × 24

## 機能

- 気象庁 CSV 読込
- 地点選択
- 月別 / 年別 / 季節別 集計
- 最大値 / 平均値 / 標準偏差
- CSV 出力
- PDF レポート出力
- グラフ表示
- 一括処理
- 実行履歴
- ログ閲覧
- 設定画面
- JMA 日別ページ取得補助
- 全国一括解析

## グラフ表示について

グラフ画面では、集計結果に応じて表示方法を切り替えています。

- 集計結果が 0 件の場合  
  - グラフは描画せず、メッセージを表示します

- 集計結果が 1 件の場合  
  - 折れ線グラフではなく棒グラフで表示します
  - 値ラベルを表示します

- 集計結果が複数件の場合  
  - 折れ線グラフで推移を表示します
  - 各点に値ラベルを表示します

### ヒストグラムの考え方

ヒストグラムは、可能であれば **イベント一覧ベース** で表示します。

- `event_df` があり、`時間` または `hours` 列を持つ場合  
  - 連続無日照イベントの時間分布として表示

- `event_df` がない場合  
  - 集計結果の値を使って表示

### 少件数データについて

sample データのように結果が 0 件または 1 件しかない場合、  
通常の折れ線やヒストグラムでは見栄えが弱くなるため、  
単一点表示やメッセージ表示に切り替えています。

## sample データについて

グラフ確認用に、複数種類の sample CSV を用意する想定です。

- `sample_jma_sunshine.csv`
  - 最小疎通確認向け

- `sample_jma_sunshine_long.csv`
  - 月別グラフを複数点で確認するための長期データ

- `sample_jma_sunshine_events.csv`
  - 複数の連続無日照イベントを発生させ、イベント分布ヒストグラムを確認するためのデータ

## 一括処理の設定CSVについて

一括処理では、元データCSVとは別に **設定CSV** が必要です。

設定CSVには、少なくとも次の列が必要です。

- 地点名
- 集計単位
- 統計値
- 開始日
- 終了日

例:

```csv
地点名,集計単位,統計値,開始日,終了日
旭川,月別,最大値,2024-01-01,2024-03-31
旭川,月別,平均値,2024-01-01,2024-03-31
旭川,年別,最大値,2024-01-01,2024-06-30
